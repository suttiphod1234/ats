Day 1: 09:00 - 10:30 (1.5 ชม.)

ทบทวนพื้นฐานและการวางโครงสร้างข้อมูล

ทบทวนการสร้าง Data Model และ DAX พื้นฐาน

รากฐานสำคัญสำหรับการวิเคราะห์ระดับสูง

ชื่อผู้สอน / หน่วยงาน

วันที่อบรม

วัตถุประสงค์การเรียนรู้

  • ออกแบบ Data Model ให้มีประสิทธิภาพสูงสุด (Star Schema)
  • เข้าใจหลักการทำงานของ DAX Engine และ Context
  • สร้าง Measure พื้นฐานที่ถูกต้องและรวดเร็ว

Data Model Design: Star Schema

แยกข้อมูลตามหน้าที่เพื่อลดความซ้ำซ้อน

  • Fact Table: เก็บตัวเลข (ยอดผลิต, ยอดขาย)
  • Dimension Table: เก็บคำอธิบาย (สินค้า, วันที่, พนักงาน)

ทำไมต้อง Star Schema?

  • Performance: ประมวลผลข้อมูลนับล้านแถวได้ในเสี้ยววินาที
  • Simplicity: ช่วยให้การเขียน DAX ง่ายขึ้นและไม่ซับซ้อน
  • Scalability: รองรับการเพิ่มเงื่อนไขการวิเคราะห์ในอนาคต

Relationships & Filtering

กฎเหล็กของการเชื่อมต่อตาราง

  • ใช้ One-to-Many (1:*) เป็นหลักเสมอ
  • Single Direction Filtering ดีกว่า Both Direction
  • ห้ามเชื่อม Many-to-Many โดยไม่จำเป็น

DAX Context: หัวใจของการคำนวณ

Context ลักษณะการทำงาน
Filter Context คำนวณตามการกรองของ Slicer หรือ Visual
Row Context คำนวณทีละแถว (เช่น ใน Calculated Column)
การเข้าใจ Context ช่วยให้ตัวเลขในรายงานแม่นยำ 100%

Measure vs Calculated Column

เลือกเครื่องมือให้เหมาะกับงาน

  • Measure: คำนวณเบื้องหลัง (แนะนำ) ไม่กินเนื้อที่ไฟล์
  • Calculated Column: สร้างแถวใหม่ในตาราง ใช้ได้น้อยลงในระดับสูง

Essential DAX Functions

ฟังก์ชันที่ต้องมีติดตัว

  • SUM / AVERAGE / COUNT: พื้นฐานการรวมกลุ่มข้อมูล
  • CALCULATE: ราชาของ DAX ใช้เปลี่ยนมุมมองการคำนวณ

ข้อควรระวัง (Common Mistakes)

  • การใช้ Flat Table ตารางเดียวที่ใหญ่เกินไป
  • ใช้ Calculated Column ทุกอย่าง (ทำให้ไฟล์อืด)
  • ไม่สร้าง Date Table แยกต่างหาก

บทสรุปและเตรียมความพร้อม

พื้นฐาน Data Model ที่ดี คือจุดเริ่มต้นของ Dashboard ที่ทรงพลัง

สไลด์ถัดไป: เจาะลึก CALCULATE ขั้นสูง
Slide 1 / 10

สรุป: การใช้ Integer Key (0, 1, 2) เชื่อมตารางเร็วกว่า String (Text) 30-50%

Fact_Production[MachineID] Dim_Machine[MachineID]
1. โครงสร้างตารางที่ดีที่สุดใน Power BI คือ?
2. Fact Table ควรเก็บอะไร?
3. ความสัมพันธ์ที่แนะนำ?
